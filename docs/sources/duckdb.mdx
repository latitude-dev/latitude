## How to configure sources

The sources files must be inside the `queries/` folder. 
<Note>To know how it works the source hierarchy, please, visit [How to configure sources](../sources/how-to-configure-sources).</Note>

## Configuration

To configure the connection with your your DuckDB database follow these steps:

<Steps>
    <Step title="Go to the sources file in your Latitude project">
        ```
            /sources/sources.yaml
        ```
    </Step>
    <Step title="Add the following code for your DuckDB connection">
    ``` yaml
    type: duckdb
    details:
        url?: string
    ```
    </Step>
    <Step title="Replace the details section with your info and save">
    </Step>
    <Step title="Done, now you can test your connection">
    </Step>
</Steps>

## Attributes

- **URL** â†’ This refers to the location of the DuckDB database file. DuckDB is an embedded database, so instead of connecting over a network, you specify the path to the database file on your local system or a shared filesystem. This path tells your application exactly where to find the DuckDB database you want to work with.

## Test connection

To test the connection got to a file in `/queries/.../your_query_file.sql` and make a query using the new connection as source.

You can check the section [SQL Syntax Basics](../queries/basics/sql-syntax-basics) to know how to query your data.
